# YoutubeSyncPlayer

複数のYouTube動画を同時に再生し、タイミングを精密に同期させるWebアプリケーションです。
Google Antigravityを用いて作成されています。

## ✨ 主な機能

*   **マルチ動画再生**: 複数のYouTube動画を追加して同時再生。
*   **精密な同期調整**:
    *   ミリ秒単位（10ms / 100ms）でのオフセット調整。
    *   **Tap Sync**: 音に合わせてタップするだけで大まかな同期設定が可能。
*   **自動ドリフト補正**:
    *   再生中の微妙なズレを検知し、自動的に同期を維持します。
    *   **Threshold（閾値）調整**: 環境や動画に合わせて、同期の「厳しさ」をリアルタイムに調整可能。
*   **共有機能 (Share)**:
    *   動画の組み合わせとオフセット設定をURLパラメータとして保存・共有できます。
    *   `?v1=ID&o1=0&v2=ID&o2=500` のような形式で再現可能。
*   **再生速度コントロール**: 全動画の再生速度を一括変更可能（0.25x 〜 2.0x）。

## 🚀 使い方

1.  **動画の追加**: 画面上部の「Add Video」ボタン、またはYouTubeのURL/IDを入力して追加します。
2.  **PrimaryとSecondary**:
    *   最初の動画が **Primary（基準）** となります。
    *   2つ目以降は **Secondary（追従）** となり、Primaryに合わせて再生されます。
3.  **同期の調整**:
    *   Secondary動画の下にある `Sync: -100 -10 ...` ボタンで、Primaryに対する遅延/先行を調整します。
    *   数値は「ミリ秒（ms）」です。
4.  **挙動がカクつく場合**:
    *   画面下部の **Threshold** を調整してください。
    *   数値を大きくする（例: 100ms）と、許容範囲が広がりカクつきが減りますが、同期精度は下がります。
    *   数値を小さくする（例: 40ms）と、厳密に同期しようとしますが、環境によっては頻繁に修正（シーク）が発生します。
5.  **共有**:
    *   「Copy Link」ボタンを押すと、現在の状態を再現できるURLがクリップボードにコピーされます。

## 🛠️ インストール & デプロイ

このアプリケーションは、サーバーサイドの処理を必要としない**静的Webサイト**です。

### ローカルでの実行
```bash
# リポジトリのクローン
git clone https://github.com/shf-spica/YoutubeSyncPlayer.git

# ディレクトリへ移動
cd YoutubeSyncPlayer

# 任意のローカルサーバーで起動（例: Python）
python3 -m http.server 8000
```
ブラウザで `http://localhost:8000` にアクセスしてください。

### サーバーへのデプロイ
Webサーバー（Nginx, Apache, Vercel, GitHub Pagesなど）の公開ディレクトリに、以下のファイルを配置するだけで動作します。

*   `index.html`
*   `script.js`
*   `style.css`

## ⚠️ 制限事項

*   **モバイル対応**: iOS/Androidのブラウザ制限により、複数の動画を同時に（音を出して）再生することはできません。タップして個別に再生を開始する必要があります。
*   **自動再生**: ブラウザのポリシーにより、最初の再生にはユーザー操作が必要です。
